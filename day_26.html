Phase 26: Building the Gizmo

Professor's Briefing

Let's make our `is_hireable` function more intelligent. It shouldn't just rely on one score. A proper hiring decision uses multiple data points. We will now build a function that takes a full candidate 'dossier' (a dictionary) and makes a decision based on multiple criteria, reflecting your own HR strategy.

The Blueprint: A Multi-Criteria Function

This function will take an entire dictionary as an argument. It can then access all the values within that dictionary to perform its logic.

Code Workbench


# This is a dossier for one candidate.
candidate = {
    "name": "Alex Ray",
    "interview_score": 91,
    "years_experience": 8,
    "skills": ["Python", "SQL", "Tableau", "Pandas"],
    "is_referral": True
}

def analyze_dossier(dossier):
    """Analyzes a candidate dictionary and decides if they are a priority hire."""

    # We can access the dictionary's values inside the function.
    score = dossier["interview_score"]
    experience = dossier["years_experience"]
    is_ref = dossier["is_referral"]

    print(f"\nAnalyzing candidate: {dossier['name']}...")
    
    # Priority candidates are high-scorers with experience OR referrals.
    # The parentheses group the 'or' condition.
    if score >= 90 and (experience > 5 or is_ref):
        return "Priority Hire. Escalate immediately."
    elif score >= 80:
        return "Solid candidate. Proceed to next round."
    else:
        return "Hold for now. Re-evaluate if pool is small."

# Let's run the analysis on our candidate.
recommendation = analyze_dossier(candidate)
print(recommendation)

Professor's Note: This is how your expertise gets encoded into a system. Your rules, your logic for what makes a good candidate, captured in a function. It's like cloning a small part of that beautiful mind.
