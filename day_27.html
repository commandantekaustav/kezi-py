Phase 27: Assembling The Mainframe

Professor's Briefing

One file is an amateur move. A professional keeps their code organized. The Mainframe of our operation will be split into two files: one file for our reusable functions (`heist_utilities.py`), and a main file (`main.py`) that imports them. This is how real applications are built—from smaller, modular components.

The Blueprint: Multi-File Projects

By keeping our functions in a separate file, we can `import` them into any other script that needs them. It makes our code clean, reusable, and easy to maintain.

The Utility File: `heist_utilities.py`


# This file only contains our functions. It doesn't run anything by itself.

def analyze_dossier(dossier):
    """Analyzes a candidate dictionary and returns a recommendation."""
    score = dossier.get("interview_score", 0)
    experience = dossier.get("years_experience", 0)
    
    if score >= 90:
        return "Priority"
    elif score >= 80 and experience > 2:
        return "Standard"
    else:
        return "Reject"

def load_candidates_from_file(filename):
    """Loads candidates from a simple text file, returns a list."""
    try:
        with open(filename, 'r') as f:
            candidates = [line.strip() for line in f.readlines()]
        return candidates
    except FileNotFoundError:
        return []

The Main Script: `main.py`


# We can import our OWN python file just like a standard module.
import heist_utilities as util

print("--- Running Main Heist Protocol ---")

# Let's create a dossier for this run.
current_candidate = {"name": "Test Subject", "interview_score": 95}

# Use the function from our imported file.
recommendation = util.analyze_dossier(current_candidate)

print(f"Recommendation for {current_candidate['name']}: {recommendation}")
