Phase 13: Coded Messages

Professor's Briefing

A good plan is useless if the next operative can't understand it. Professionals leave notes. They document their work. In Python, there are two ways to do this: **comments** for brief, informal notes, and **docstrings** for formal, official documentation of a function's purpose.

The Blueprint: Comments and Docstrings

Comments (`#`)

A comment starts with a hash symbol `#`. The Python interpreter completely ignores anything on the line after the `#`. They are informal messages from the coder to themself or their teammates, explaining *why* a piece of code is written a certain way.

Docstrings (`"""..."""`)

A docstring is a special type of string enclosed in triple-quotes that sits on the very first line after a function definition. It is the function's official documentation. It explains what the function does, what arguments it expects, and what it returns. Professional code is full of docstrings.

Code Workbench


# This is a comment explaining the purpose of the next function.
# It's intended to check if a candidate's skills match the job requirements.

def analyze_candidate_skills(candidate_skills, required_skills):
    """Checks for required skills and calculates a match percentage.

    This function is a core part of our automated screening process.
    It takes a list of the candidate's skills and a list of required
    skills, and returns a percentage of how many requirements are met.

    Args:
        candidate_skills (list): A list of strings, each a skill.
        required_skills (list): A list of strings, each a required skill.
    
    Returns:
        float: The percentage of required skills the candidate possesses.
    """
    # Use a loop to count the matches (we'll cover this logic later)
    matches = 0
    for skill in required_skills:
        if skill in candidate_skills:
            # Found a match!
            matches += 1
    
    # Avoid division by zero if there are no required skills.
    if len(required_skills) == 0:
        return 100.0 # If nothing is required, it's a perfect match.
    
    return (matches / len(required_skills)) * 100

# We don't need to run this, just understand the documentation.
