Phase 17: Planting Disinformation

Professor's Briefing

Intercepting comms is only half the job. We also need to plant our own messages—to create reports, logs, and new dossiers. Today you learn to **write to files**. This is how a program leaves a permanent record of its work, long after the program itself has finished running.

The Blueprint: Writing to a File

The process is similar to reading, but we use write mode (`'w'`) or append mode (`'a'`).
• `'w'` (Write): Creates a new file or **overwrites** an existing file completely.
• `'a'` (Append): Creates a new file or adds new content to the **end** of an existing file.

Code Workbench


# Candidate data we've collected in our script
candidates_to_shortlist = ["Priya Sharma", "Aisha Khan"]
rejected_candidates = ["David Chen"]

report_filename = "hiring_report.txt"

# Let's use 'w' to create and write the report header.
# This will overwrite the file if it already exists.
with open(report_filename, 'w') as f:
    f.write("--- CONFIDENTIAL HIRING REPORT ---\n")
    f.write("Generated automatically.\n")
    f.write("--------------------------------\n\n")

# Now, let's use 'a' to append the short-listed candidates
# without erasing the header.
with open(report_filename, 'a') as f:
    f.write("SHORTLISTED FOR FINAL INTERVIEW:\n")
    for candidate in candidates_to_shortlist:
        f.write(f"- {candidate}\n") # \n adds a new line

# Let's append the rejected list as well.
with open(report_filename, 'a') as f:
    f.write("\nREJECTED AT THIS STAGE:\n")
    for candidate in rejected_candidates:
        f.write(f"- {candidate}\n")

print(f"Report '{report_filename}' has been successfully generated.")

Professor's Note: `'w'` erases everything and starts fresh, `'a'` adds to what's already there. Your presence in this operation feels like you've completely overwritten the old plan. A welcome change.
