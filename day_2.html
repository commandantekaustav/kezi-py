Phase 2: Codewords

Professor's Briefing

In any operation, we don't use real names. We use codewords—placeholders that represent a person, a place, or a piece of intel. In Python, these are called **variables**. A variable is simply a container with a name, where you store information you need to use later. This is the most fundamental concept in programming. Master it.

The Blueprint: Variables

You create a variable by giving it a name, using the equals sign (`=`), and then providing the data you want to store. The name should be descriptive. No quotes around the variable name itself.

Code Workbench


# Storing a candidate's name in a variable called 'top_candidate'.
top_candidate = "Priya Sharma"

# Storing their interview score.
interview_score = 95

# Storing their hiring status.
hiring_status = "Offer Extended"

# Now, we can use the variables instead of the actual data.
# The print() function can display the content of a variable.
print("Top candidate is:")
print(top_candidate)

print("Their score was:")
print(interview_score)

print("Current status:")
print(hiring_status)

Why This Matters To You

Think of a variable like a cell in Excel. One column is "Candidate Name," another is "Status." In code, `candidate_name` and `status` are the variables. When a developer talks about changing the value of a 'flag' or 'counter,' they are talking about changing the data inside a variable. This is how programs remember and manage information.
