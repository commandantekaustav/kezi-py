Phase 6: The Target List

Professor's Briefing

An operative is useless if they can only track one target at a time. We need to manage lists of targets, lists of equipment, lists of contacts. In Python, the most fundamental way to store an ordered collection of items is a **list**. It is your digital target list.

The Blueprint: Python Lists

A list is a container that holds multiple items in a specific order. You create a list using square brackets `[]`, with items separated by commas.

Code Workbench


# A list of candidate names (strings)
candidate_pipeline = ["Priya Sharma", "David Chen", "Aisha Khan"]
print(f"Current pipeline: {candidate_pipeline}")

# Accessing an item by its index. Python starts counting from 0.
# The first item is at index 0.
first_candidate = candidate_pipeline[0]
print(f"First candidate to interview: {first_candidate}")

# Adding a new candidate to the end of the list with .append()
candidate_pipeline.append("Maria Garcia")
print(f"Updated pipeline: {candidate_pipeline}")

# Removing a candidate from the list with .remove()
candidate_pipeline.remove("David Chen")
print(f"After interview: {candidate_pipeline}")

# Checking how many candidates are in the pipeline with len()
num_candidates = len(candidate_pipeline)
print(f"Number of candidates remaining: {num_candidates}")

Why This Matters To You

Lists are everywhere in programming. A list of users, a list of products, a list of search results. Every time you see an ordered collection of items on a screen, a list (or a similar structure) is powering it behind the scenes. This is how developers manage data in bulk.
