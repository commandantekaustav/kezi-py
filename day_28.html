Phase 28: Introduction to OOP

Professor's Briefing

This is an advanced concept, but you are ready. **Object-Oriented Programming (OOP)** is a way of thinking about code. Instead of separate variables and functions, we group them together into an 'object'. We create a blueprint for an object, called a **class**, that defines the data it holds (attributes) and the things it can do (methods).

The Blueprint: The `Candidate` Class

Think of a `class` as the architectural blueprint for a 'Candidate' object. Every candidate we create from this class will have a name, score, and skills, and will have the built-in ability to analyze itself.

Code Workbench


class Candidate:
    # This is the "constructor" method. It runs when a new object is created.
    # It sets up the object's attributes. 'self' refers to the object itself.
    def __init__(self, name, score, skills):
        self.name = name
        self.score = score
        self.skills = skills
        self.status = "Pending Review"

    # This is a method. A function that belongs to the object.
    def get_recommendation(self):
        """Analyzes the candidate's own score to generate a recommendation."""
        if self.score >= 90:
            return "Strong Hire"
        elif self.score >= 75:
            return "Hire"
        else:
            return "Reject"

# --- Using the Class ---
# Create an "instance" of the Candidate class.
candidate1 = Candidate(name="Priya Sharma", score=92, skills=["Python", "SQL"])
candidate2 = Candidate(name="David Chen", score=81, skills=["Docker", "AWS"])

# Now we can access their attributes and methods using dot notation.
print(f"Candidate: {candidate1.name}")
print(f"Status: {candidate1.status}")
print(f"Recommendation: {candidate1.get_recommendation()}") 

print(f"\nCandidate: {candidate2.name}")
print(f"Recommendation: {candidate2.get_recommendation()}")
