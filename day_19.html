Phase 19: Bringing in Specialists

Professor's Briefing

We don't build every tool from scratch. Sometimes, we need an outside specialist—an expert in explosives, a master of disguise. In Python, this means **importing modules**. A module is a file containing pre-written Python code (functions and variables) that we can bring into our own plan to give us new capabilities.

The Blueprint: Importing Modules

Python comes with a huge "standard library" of built-in modules. You use the `import` keyword to gain access to the functions inside them. Today, we'll import the `random` module, a specialist in chance and unpredictability.

Code Workbench


# Bring in the 'random' specialist.
import random

candidate_pool = ["Priya Sharma", "David Chen", "Aisha Khan", "Maria Garcia", "Omar Hassan"]

print("--- Randomly selecting a candidate for spot-check interview ---")

# We can now use functions from the random module.
# To call a function from a module, we use module_name.function_name()
# random.choice() selects one random item from a list.
selected_candidate = random.choice(candidate_pool)

print(f"The selected candidate is... {selected_candidate}")


print("\n--- Generating a random 4-digit security code ---")
# random.randint(a, b) returns a random integer between a and b (inclusive).
security_code = random.randint(1000, 9999)
print(f"New code for {selected_candidate}: {security_code}")

Why This Matters To You

No professional developer works alone. They constantly `import` modules. They import `pandas` for data analysis, `requests` for web communication, `datetime` for handling dates. When you see `import` at the top of a Python file, it's the developer listing the outside specialists they've brought in for the mission. It tells you immediately what kind of operation they are running.
