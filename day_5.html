Phase 5: The Interactive Dossier

Professor's Briefing

A plan shouldn't be static. Sometimes, you need to provide fresh intel while the operation is running. The `input()` function allows you to do just that. It pauses the program and waits for the user to type something and press Enter. It turns a one-way script into a two-way conversation.

The Blueprint: The `input()` Function

The `input()` function prompts the user for information. The text you put inside the parentheses is the prompt the user sees. The function then "returns" whatever the user types as a **string**.

Code Workbench


# Ask for the candidate's name and store it in a variable.
candidate_name = input("Enter candidate's full name: ")

# Ask for the role they are applying for.
role = input("Enter the role they are applying for: ")

# Ask for their years of experience.
# Remember, input() returns a STRING.
experience_str = input("Enter years of experience: ")

# Create a summary message.
# The 'f' before the string creates an "f-string",
# which lets you embed variables directly in curly braces {}.
summary = f"Processing dossier for {candidate_name}, applying for {role} with {experience_str} years of experience."

print("\n--- DOSSIER SUMMARY ---")
print(summary)
print("---------------------\n")

# To use experience_str as a number, we must convert it.
# This is called 'casting'. We cast the string to an integer.
experience_int = int(experience_str)
next_year_experience = experience_int + 1
print(f"Next year, they will have {next_year_experience} years of experience.")

Why This Matters To You

This is the heart of interactivity. Every form you fill out online, every time you type in a search bar, a function similar to `input()` is capturing your text. Understanding this helps you see that software isn't magic; it's a series of conversations between the user and the code.
